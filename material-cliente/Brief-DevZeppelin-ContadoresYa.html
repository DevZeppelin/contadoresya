Correcciones en servidor / CMS — Antes de subir los HTML
Crítico: Estas correcciones deben aplicarse en el servidor antes o simultáneamente con la subida de los HTML. Sin ellas, el sitio sigue sirviendo URLs duplicadas que dañan el SEO.
🔴 Redirección 301: no-www → www CRÍTICO
Problema detectado: El sitio responde tanto en contadoresya.ar (sin www) como en www.contadoresya.ar. Google las trata como dos sitios distintos, dividiendo la autoridad. Todos los canonicals nuevos apuntan a www.

Solución: Forzar redirect 301 permanente de no-www a www en .htaccess (Apache) o en la configuración de Nginx/hosting.
# .htaccess — Apache RewriteEngine On RewriteCond %{HTTP_HOST} !^www\. [NC] RewriteRule ^(.*)$ https://www.%{HTTP_HOST}/$1 [R=301,L] # Nginx (nginx.conf) server { listen 80; server_name contadoresya.ar; return 301 https://www.contadoresya.ar$request_uri; }
🔴 Forzar HTTPS en todo el sitio CRÍTICO
Problema: Si el sitio responde también por HTTP, Google puede indexar versiones duplicadas y el navegador muestra "Sitio no seguro".

Solución: Redirigir todo el tráfico HTTP → HTTPS con 301 permanente.
# .htaccess — Apache (combinar con el bloque www arriba) RewriteCond %{HTTPS} off RewriteRule ^(.*)$ https://www.contadoresya.ar/$1 [R=301,L]
🟠 Eliminar páginas /contadores-en-cordoba y /contadores-en-buenos-aires y redirigir ALTO
Problema: Estas dos páginas existían como plantillas casi idénticas a /contadores-en-mendoza, generando contenido casi-duplicado. El Estudio no ofrece servicios en esas provincias.

Solución: Eliminar los archivos del servidor y configurar redirect 301 hacia home.
# .htaccess Redirect 301 /contadores-en-cordoba https://www.contadoresya.ar/ Redirect 301 /contadores-en-buenos-aires https://www.contadoresya.ar/
🟠 Canonical: actualizar todas las canónicas a versión www ALTO
Problema detectado: El sitio actual tenía canonicals apuntando a https://contadoresya.ar/ (sin www), mientras el sitio servía con www. Inconsistencia que confunde a Google sobre la URL preferida.

Solución: Los nuevos HTML ya tienen canonicals corregidos (https://www.contadoresya.ar/página). Solo verificar que el CMS no sobreescriba las etiquetas canonical con su propia lógica.
🟡 Verificación Google Search Console para www.contadoresya.ar MEDIO
Acción: En Google Search Console, verificar que la propiedad preferida sea https://www.contadoresya.ar (con www) y solicitar reindexación de las páginas modificadas una vez subidas. También revisar si ya hay una propiedad para https://contadoresya.ar (sin www) y marcarla como propiedad secundaria.

Instrucciones de implementación — Orden de ejecución
Importante: Seguir el orden indicado. Los pasos 1-3 son de servidor y deben resolverse antes de subir los archivos HTML.
Configurar redirect 301 no-www → www + HTTP → HTTPS
Modificar .htaccess (Apache) o nginx.conf. Verificar que contadoresya.ar y http://contadoresya.ar redirijan correctamente a https://www.contadoresya.ar. Testear con herramienta de redirección antes de continuar.
Eliminar o redirigir /contadores-en-cordoba y /contadores-en-buenos-aires
Borrar los archivos del servidor y agregar redirect 301 de ambas URLs hacia home. Verificar que devuelvan 301 y no 404.
Verificar que el CMS no sobreescriba las etiquetas <title> y canonical
Algunos CMS (WordPress, Joomla, etc.) insertan sus propios title y canonical, sobreescribiendo los del HTML. Desactivar esa función para las páginas que se van a reemplazar, o subir los HTML como archivos estáticos fuera del CMS.
Subir los 8 archivos HTML a sus rutas correspondientes
Respetar exactamente las URLs indicadas en la Sección 1. Verificar que cada página devuelva código 200 y que el title sea el correcto (inspeccionar con F12 → Elements).
Actualizar el mapa del Google Maps en /contadores-en-mendoza
El iframe de Google Maps en contadoresya-contadores-en-mendoza.html tiene un embed genérico como placeholder. Reemplazarlo con el mismo embed que ya está funcionando en el sitio actual. El código del iframe se obtiene en Google Maps → Compartir → Incorporar un mapa. Verificar que muestre la ficha "Contador Público Matriculado en Mendoza - Dario Vallinas".
Actualizar y enviar el sitemap.xml a Google Search Console
Reemplazar el sitemap existente con el nuevo (ver Sección 2). Enviarlo en Search Console → Sitemaps → Agregar nuevo sitemap. Si ya existe una propiedad para no-www, gestionarla como propiedad secundaria.
Solicitar reindexación de las páginas modificadas en Search Console
Ir a Search Console → Inspección de URL → pegar cada URL → Solicitar indexación. Hacerlo en orden: home, /deudas-arca, /monotributo, /impuestos, /sueldos, /certificacion-de-ingresos, /contadores-en-mendoza, /estudio-contable.
Verificar render de Schema.org en todas las páginas
Usar la herramienta de prueba de resultados enriquecidos de Google (search.google.com/test/rich-results) para validar que los schemas FAQPage, AccountingService y BreadcrumbList se detecten correctamente en cada URL.
Validar en mobile y verificar velocidad con PageSpeed Insights
Abrir cada página en PageSpeed Insights (pagespeed.web.dev) y verificar que el puntaje mobile supere 80. Los archivos son HTML estático con CSS inline, lo que favorece la velocidad.

Contexto del proyecto: Este sitio es el tercero de un ecosistema de tres dominios del Estudio Contable Vallinas. Los otros dos (contador-en-mendoza.com.ar y estudio-contable-mendoza.com.ar) ya fueron actualizados en una entrega anterior. El objetivo central de este rediseño fue eliminar la canibalización de keywords entre los tres dominios, asignando a cada uno un rol y una intención de búsqueda distintos:

• contador-en-mendoza.com.ar → "contador en Mendoza" (presencial, local, orgánico)
• estudio-contable-mendoza.com.ar → "estudio contable Mendoza" (Google Ads + conversión)
• contadoresya.ar → "deuda ARCA", "monotributo Argentina", "contador online" (nacional, orgánico, urgencia)
